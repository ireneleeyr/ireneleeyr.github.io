Overview

Project Summary

This project developed an AI-powered application that monitors a student's study condition in real time using a standard webcam. Using a convolutional neural network trained with Google Teachable Machine, the system classifies the student's state into three categories: focused, sleeping or distracted, and absent.

The program captures each webcam frame, preprocesses it, and feeds it into the model to generate a prediction with a confidence score — providing instant feedback on study behavior. The dataset consisted of 150 images per class (450 total), carefully curated to remove blur, duplicates, and bias from lighting and background variation.

Motivation & Problem Definition

Why Does This Matter?

😴

The Problem

It is hard to objectively determine whether we are truly focusing or not. Students frequently drift into distraction or sleep without realising it, reducing study efficiency significantly.

🤖

The Approach

A study monitoring system that uses computer vision to automatically detect whether the user is concentrating, sleeping, or absent — and provides immediate, objective feedback.

🎯

The Goal

Provide real-time feedback on study state to promote better focus and improve study efficiency — through a lightweight, accessible AI application built with Python and OpenCV.

Dataset & Methods

How It Works

Dataset Collection & Cleaning

Images were captured via webcam in varied conditions: different postures, angles, and lighting. Initially 200 images per class were collected, then cleaned down to 150 each.


  • Removed blurry and duplicated images
  • Balanced dataset across 3 classes (150 × 3 = 450)
  • Controlled for lighting and background bias
  • Class 0 — Focused: proper study posture
  • Class 1 — Sleeping: eyes closed / head down
  • Class 2 — Absent: user not visible

Model & Libraries

Trained using Google Teachable Machine (MobileNet-based CNN), exported as Keras .h5. Integrated into Python with the following libraries:


  • OpenCV: real-time webcam capture, resize, display
  • TensorFlow/Keras: load model, run prediction
  • NumPy: convert frames to arrays, normalize

Input: 224×224×3 RGB · Normalize: (pixel/127.5)−1 · Output: argmax over 3 classes

Actual Code
# ── AI Study Concentration Detector ── Yerin Lee, 20260861 ── import cv2 import numpy as np from tensorflow.keras.models import load_model model = load_model("keras_model.h5", compile=False) class_names = open("labels.txt", "r").readlines() # focused / sleeping / absent cap = cv2.VideoCapture(0) while True: ret, frame = cap.read() if not ret: print("웹캠 오류") break image = cv2.resize(frame, (224, 224)) image = np.asarray(image, dtype=np.float32).reshape(1, 224, 224, 3) image = (image / 127.5) - 1 # normalize to [-1, 1] prediction = model.predict(image) index = np.argmax(prediction) # highest confidence class class_name = class_names[index] confidence = prediction[0][index] print("Class:", class_name.strip(), "Confidence:", confidence) cv2.imshow("Webcam", frame) if cv2.waitKey(1) & 0xFF == ord('q'): break cap.release() cv2.destroyAllWindows()

Discussion & Applications

What We Learned & What's Next

💪 Strengths

  • Simple, practical idea with clear real-world utility
  • Real-time detection with low-latency inference
  • Clear integration of AI model with Python program
  • Careful data curation to reduce bias (lighting, background)

⚠️ Limitations

  • Small dataset (150 images per class) limits generalization
  • Sensitive to lighting conditions and camera angle
  • Some confusion between "focused" and "sleeping/distracted"

🧠 Key Learnings

  • How to train and export a CNN model with Teachable Machine
  • Real-time AI application development with Python + OpenCV
  • Data quality (cleaning, balancing, bias control) is critical for model performance
  • Iterative testing and improvement are essential steps in any ML project

🚀 Future Improvements

  • Collect more diverse data across people and environments
  • Separate "sleeping" and "distracted" into distinct classes
  • Add a warning alert system and study statistics dashboard
  • Deploy as a standalone desktop app for self-regulated learning
